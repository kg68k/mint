・HUPAIR への対応について

	　HUPAIR とは、
		Human68k User Program Argument Interface Regurations
		(Human68k ユーザ・プログラム引数インターフェース規定)
	の略称で、Human68k 上のユーザ・プログラム間での引数列の引き渡し、
	受け取りの方法を規定したものです。

	　mint v3 以降は HUPAIR に完全対応しています。

	　mint を子プロセスとして起動して引数を渡した場合、引数は HUPAIR
	に従ってデコードされます。例えば、
		mint -s ".mintrc'00"
	というコマンドを実行すると(fish.x、COMMAND.X どちらでも)
		引数 0	mint
		引数 1	-s
		引数 2	.mintrc'00
	という引数列として解釈します。

	　HUPAIR に従った引数のデコードは、与えられた引数が HUPAIR でエ
	ンコードされていなくても、常に実行されます(そのため、上記の例は
	COMMAND.X でも同じ結果になります)。

	　mint が子プロセスを起動して引数を渡す場合、引数は HUPAIR に従
	ってエンコードされます。ただし、子プロセスが HUPAIR に対応してい
	ない場合は HUPAIR に従ったエンコードは行わず、単純に引数列を繋げ
	ただけの文字列として渡します。

	　HUPAIR に対応していない子プロセスに対しては、引数を正しく渡す
	ことが出来ない場合があります。また、実装上の理由で一度 HUPAIR エ
	ンコードしてからデコードして戻したものを子プロセスに渡すため、若
	干の処理時間がかかります。このように、HUPAIR に対応していないプ
	ログラムを使用する場合は不利な点があるので、出来る限り HUPAIR に
	対応したプログラムを使用するか、プログラムを HUPAIR 対応に修正す
	ることをお勧めします。


・mint における環境変数領域の扱いについて

	　通常、mint は親プロセスから渡された環境変数領域をそのまま使用
	しているため、&setenv や &unsetenv を実行すると、親の環境変数領
	域にもその結果が反映されます。もちろん、mint からの子プロセスに
	も反映されます。

	　ただし、mint を -E オプションを指定して起動した場合は環境変数
	領域を独自に確保するので、親プロセスの環境変数領域が影響を受ける
	ことはありません。

	　また、親プロセスに環境変数領域を保護する機能が備わっている場合
	もも影響を受けません。該当するプログラムにはシェルの fish.x など
	があります(なお、fish のシェル変数 envcmds に mint.x を指定する
	と、保護機能が抑制されるので影響を受けるようになります)。


・.MAG 形式のタイトル画像作成時の推奨事項

	　以下に述べていることはあくまで推奨であって、強制ではありません。
	また、現時点では正式決定ではなく、案とします。「余計なお世話だ」
	とか「～～のガイドラインがないと困る」といった点があれば是非報告
	して下さい。

	　8 色画像はパレット 0～7 を使用し、パレット 8～15 には 0～7 と
	同じ色を設定しておいて下さい。4 色画像はパレット 0～3 を使用し、
	パレット 4～7、8～11、12～15 にはそれぞれ 0～3 と同じ色を設定し
	ておいて下さい。こうしておかないと色数の自動判別が出来ません。

	# いずれの場合にも、可能なら 8 色フラグを立てておくとなお良いで
	# す。それが可能なツールは見たことありませんけど…

	　パレット 0 は黒か、使用している色の中でもっとも暗い色にして下
	さい。黒は不透明な黒ではなく透明な黒になります。パレットに余裕が
	ある時は、パレット 0 の黒とは別に、.MAG ファイルにおける値を
	($00 $00 $08) にした疑似的な黒を設定し、画像内の黒には全てそちら
	を使うとよいかもしれません。

	　パレット 3 は白か、使用している色の中でもっとも明るい色にして
	おくと、テキスト画面に残っている文字が読みやすくて良いと思います。

	　画像のサイズと表示座標は、当たり前ですが 768x512 に収まる範囲
	にしましょう。ファンクションキー表示行や左上のカレンダー/時刻表
	示にもかぶらないようにすべきです。また、コピー中などに使うものは
	コンソールにかぶらないようにあまり下に表示しない方が良いと思いま
	す。もっとも、表示座標はオプションで変更できるのでたいした問題で
	はありませんが。

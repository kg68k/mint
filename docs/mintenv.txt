		[ mint 環境変数マニュアル ]


------------------------------------------------------------------------

　madoka 内で環境変数を設定するときは &setenv foo 0 などと記述する。参照
する場合は $foo とする。環境変数の内容を消去する場合は &unsetenv foo と
する。

------------------------------------------------------------------------

HOME

　定義ファイル `.mint' のパスを設定する。

MINTSHELL

　mint が使用するコマンドシェルのパスとファイル名、及び起動オプションを
設定する。`MINTSHELL' が定義されていない場合は `COMMAND.X' が使用される。

MINTTMP

　mintarc が一時ファイルを開くディレクトリを設定する。これが存在しない場
合は環境変数 `temp' が使用される。

temp

　環境変数 `MINTTMP' が存在しない場合、その変わりに参照される。これが存
在しない場合は環境変数 `TEMP' が使用される。

TEMP

　環境変数 `MINTTMP' `temp' のいずれも存在しない場合、その変わりに参照さ
れる。

MINTSLASH

　mint が使用するパスの区切り文字 `\' または `/' を設定する。これが存在
しない場合は環境変数 `SLASH' が使用される。

SLASH

　環境変数 `MINTSLASH' が存在しない場合、その変わりに参照される。

MINTHIS

　レジュームファイルのディレクトリ及びファイル名を設定する。設定しないと
レジュームファイルは使用されない。

MINTHIS3

　v3 以降専用の環境変数で、`MINTHIS' に先立って参照される。

MINTRC

　環境変数 `HOME' を設定しない場合、または $HOME/.mint が存在しない場合
に読み込む mint の定義ファイルのディレクトリ及びファイル名を設定する。

MINTRC3

　v3 以降専用の環境変数で、`MINTRC' に先立って参照される。

MINTTITLE

　起動時に表示する .mag/.mit ファイルのディレクトリ及びファイル名を設定
する。設定しないと起動時に .mag/.mit ファイルは表示されない。

MINTMES

　mint が表示する &m_mes をこの環境変数によって替えることが出来る。

MINTMES2

　この環境変数を定義していると、現在カレントパス表示部分に、この環境変数
に定義した文字列が変わりに表示される。

MINTCDPATH

　ここにディレクトリ群を登録しておくと、&cd でフルパスを指定しなくともそ
のディレクトリを開くことが出来る。複数のディレクトリを連ねるときは環境変
数 path と 同じように `;' で区切る。

MINTWORDCHARS

　readline complete で単語として認識する特殊文字を指定する。ASCII 英数字
及び SHIFT JIS 文字は常に単語として認識される。この環境変数を指定しない
場合、`*?_-.[]~=' の設定が行われているものとする。

mintpath1

　mint が起動したときに開く左ウィンドウのパス名を示す。mint が終了したと
き、システム変数 %prfl が 0 ならば、この環境変数に終了時の左ウィンドウの
パス名が格納される。

mintpath2

　mint が起動したときに開く右ウィンドウのパス名を示す。mint が終了したと
き、システム変数 %prfl が 0 ならば、この環境変数に終了時の右ウィンドウの
パス名が格納される。

・著作権の扱い

　Madoka INTerpreter (以下 mint) は、STF というファイラを前身とし、更に
その STF は TF というファイラを前身としています.

　TF が生まれ、STF に進化し、mint として生れ変わってきた間に、多くの人々
の手を渡ってきました。この場合、著作権の扱いは非常に複雑になってしまうの
ですが、幸いなことに歴代の作者の方々は「委任」という形で権利関係をも引き
継いできました。

# もちろん著作者人格権は別です。

　TF ～ STF ～ mint の委任の流れについては、mint v2 に収録されていた
COMMIT というファイルを転載したので、そちらを見て下さい。また、mint v2
での権利関係は、同じく mint.txt を見て下さい。

　さて、ここで当然 mint v2 の作者である Leaza 氏から、私(立花)への委任状
があるべきなのですが、実際の委任は電話一本で比較的アバウトに行われてしま
ったため、書面には残っていません。

　とにかく、電話での話し合いにおいて、そのあたりを全て任せていただくとの
ことでしたので、そういうことにしておいて下さい。


・ソースコード公開

　当時私はオープンソースにかぶれていたため(今では GNU 信奉者ですが:-)、
電話で「最終的にはソースコードを公開したい」という意思を伝えたのですが、
あまり奇麗に記述されていないので出来ないという結果になりました。

　受け取ったソースコードを見てみたところ、確かにそれほど奇麗ではありませ
ん。多くの作者の手を渡ってきたこともあり、これは仕方のないことです。

　しかし、これだけのソフトウェアなのにソースコードを公開できないのは惜し
いことなので、バージョンアップを行う際にソースコードを整頓していくことに
しました。「奇麗」であれば問題はありませんので。

　なお、整頓したソースコードを公開することは電話では話していませんが、結
果的にほとんどの記述が私によるものに置き換わったことと、非公開の理由であ
る「奇麗でない」というのが消失したこと、mint の一切の扱いを委任されたこ
とで、問題ないと考えます。

　整頓の結果は src ディレクトリで見ることが出来ます。もし奇麗でないと感
じたら、それは私の技術不足ですので、全て私の責任です。


・そして GPL2 へ…

　ソースコード公開にあわせ、ライセンスを『GNU GENERAL PUBLIC LICENSE
Version 2』(以下 GPL2)に変更しました。

　もともと mint はフリーソフトウェアでしたので、それほど違いはないので、
変更自体には問題ない筈です(委任されてますし)。現在フリーソフトウェアのラ
イセンスでは主流のものですし、ソースコードや権利関係の扱いを明確にしてい
るという点では新生 mint にふさわしいライセンスでしょう。


・GPL3 への変更

4.0.0 で追加した phantomx.s が GPL3 のため、全体としてのライセンスを
GPL3 に変更しました。
